1007 --block element vs inline element

#___ block lvl element
---------------------------
will occupy the full horizontal width of its parent element & vertical height
each block element will have a new line b4 & after

< div> < /div> --> means divided element --> used to divide content into sections
< form> < /form>
< h1> < /h1> - < h5> < /h5>

#___ inline elements
---------------------------
< a href="index.html"> < /a>
< img src="photo.png">
< input>
< label> < /label>
< b> < /b> for bold
< i> < /i> for italics
< em> < /em> for emphasis
< span> < /span>

#___ example of div with span inside --> block w/ inline inside the block element
< div>
... < span> < /span>
... < span> < /span>
... < span> < /span>
< /div>

#___ example w/ div changing the span to div -- which will go from across one line, to 3 seperate lines
< div>
... < span> < /span>
... < div> < /div>
... < span> < /span>
< /div>

#--- adding the link for the stylesheet in the head section of html
-----------------------

< /head>

}

# element w/ class selector ex1 --> < p class="intro"> < /p>
-------------------
p.intro {
.... margin: 2px;
}

# descendant selectors --> used when need to select elements contained w/in another selector
< div id="blog">
.... < h1> latest news < /h1>
.... < div>
........ < h1> todays weather < /h1>
........ < p> < /p>
.... < /div>

CSS
#blog h1 {
.... color: blue
}

#blog div h1 {
.... color: blue;
}

# child selectors --> more specific than descendant selectors --> only select elements that are immediate
children of the parent --> will select the #blog and then all the < h1> elements in it
---------------------
#blog > h1 {
.... color: blue;
}

#___ :hover pseudo-class
