#__6__ Presentation layer here
-----------------
deals w/ data formatting, encryption, decryption, data presentation
converts data into a usable format for the receiving app.
establishes a common format for the data so it can be understood by both sender/receiver
Data unit: data
Protocols: HTTP: port 80, SMTP: port 25

#__5__ Session layer
-----------------
responsible for managing communication sessions / states between applications
Data unit: data
Protocols: HTTP: port 80, SMTP: port 25
SSL (secure socket layer) --encrypts data, authenticates data origins, ensures data integrity

DTLS (datagram communication security protocol based on TLS)

#__4__ Transport layer
-----------------
establishes connections between devices & can ensure reliable transport between the two
responsible for delivery of data between apps on seperate hosts
assigns the port numbers to identify types of data during transmission
Data unit: Segments
Protocols: TCP: protocol 6, UDP: protocol 17

#__3__ Network access layer
-----------------
selects the best path to route data packets over a network
responsible for transmitting data between nodes on the same network
addresses devices w/ IPv4 & IPv6 protocols
Data unit: packets
Protocols: IPv4, IPv6
IPSec -- protocol & algorithm suite that secures data transferred over public networks like internet
|--> originally contained the ESP & AH protocols
|--> can protect sensitive data & VPNs providing tunneling to encrypt data transfers
|--> can encrypt at the app layer 7.

ESP (encapsulating security payload) --> encrypts data & provides authN
AH (authN header) --> offers anti-replay capabilities & protects data integrity

#__2__ Data Link layer
-----------------
Checks the data is ready to send making sure its reliable & error free
Data unit: frames
Protocols: Ethernet, WiFi

#__1__ Physical layer
-----------------
converts the data into electrical signals for transmission thru cables or wireless thru air
Data unit: bits
Protocols: Ethernet, WiFi

TCP / IP (transmission control protocol / internet protocol)

#____ how TCP/IP works
--------------------
1 --> break data into packets
2 --> sending packets
3 --> receiving packets
4 --> checking for errors

for each segment of data:
sequence number --> order of the packets
acknowledgment number (ACK) --> used to check if packages arrived
