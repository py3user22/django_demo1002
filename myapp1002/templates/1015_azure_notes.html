Hypervisors and VMs

Hypervisor --> virtualization software that is run on the physical host machine

#____ 2 types:
--------------
type 1 -- bare meal hypervisors
|.. directly installed on host machines hardware
|.. ex1. VMware, ESXI, Microsoft Hyper V, Citrix Xen server
|.... WSL (windows subsystem for Linux) -- uses hyper V architecture

type 2 -- hosted hypervisors
|.. run ontop of a host OS
|.. mainly used for desktop virtualization, test env., personal use
|.. ex1. Oracle VirtualBox, VMware, Workstation, Parallels desktop

VMs and containers demo

#____ VM (virtual machine)
--------------
a.k.a. virtual server --> software system that works like a single virtual pc
can split the physical machine or server into multiple VMs
each sharing resources such as RAM, CPU, and network.
ex1. VMware Workstation, VirtualBox, Hyper-V

#____ Containers
------------------------
lightweight software package providing isolated env. for running apps
each container shares the underlying OS w/ the host machine
each container has its own file system, network, app space
ex1. Docker, Kubernetes, Podman

XaaS (anything as a Service)

#____ SaaS (software as a Service)
------------------
users connect to and use cloud based apps over the internet
users dont install any software and sign in on browser to use the hosted software
ex1. Netflix, Microsoft 365, Spotify...
customer responsibility:
Information & data
Devices (mobile & pc)
Accts & identities
shared: Identity & directory infrastructure

#____ PaaS (Platform as a Service)
------------------
is a complete development and deployment environment in the cloud
co. uses to create & publish new programs or apps, w/o having to manage the infrastructure
ex1. Microsoft Azure, AWS lambda, Google app engine
customer responsibility:
Information & data
Devices (mobile & pc)
Accts & identities
shared:
Identity and directory infrastructure
Applications
Network controls

#____ IaaS (Infrastructure as a Service)
------------------
cloud computing service that offers computing, storage, and networking resources on demand
client will purchase, install, configure, manage their own software, OS and apps
cloud provider will manage the infrastructure
User can choose the OS, and determine the power of CPU's to use
customer responsibility:
Information & data
Devices (mobile & pc)
Accts & identities
Identity and directory infrastructure
Applications
Network controls
Operating system
