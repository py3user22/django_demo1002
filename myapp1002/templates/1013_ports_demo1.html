#_____ Ports --> used to identify protocols
------------------
used to identify different types of data during transmission
assigned w/in the transport layer
port range: 0 - 65535

#_____ Ports
------------------
Protocols: pick which want to use for data transmissions
_____ port
HTTP:___ 80
HTTPS:_ 443
FTP (file transfer protocol):
|____ 21, 20
Telnet:
|____ 23
SSH (secure shell):
|____ 22
POP3 (post office protocol 3):
|____ 110
|___> used to receive email

IMAP4 (interactive mail access protocol):
|____ 143
|___> used to receive email

SMTP (simple mail transfer protocol):
|____ 25
|___> used to send email
|__> multiple message exchanges:
|__> HELO --used to create the SMTP connection between client / server
|__> MAIL FROM --who is sending the email
|__> RCPT TO --reports the emails recipient
|__> DATA --prepare & initiate the transfer of info between client /server
|__> RSET -- reset the connection & clear all earlier transfer data w/o closing the SMTP connection
|__> QUIT -- ends connection

# the destination port is always the protocol being used

#_____ Network protocols are grouped into 3 categories

Network communication protocols
|____>

Network security protocols
|____> SSL (secure socket layer) -> standard encryption & security protocol
|
|____> TLS (transport layer security) -> replaces SSL & provides better security.
|___> based on IETF standard

HTTPS (hyper text transfer protocol secured)

SSH (secure shell) --> cryptographic network security protocol

Network mgmt protocols
-----------------------
used to monitor and manage devices and detect faults on performance issues

SNMP (simple network mgmt protocol) --> allows for the collection of data from devices on your network
|__> and the mgmt of those devices

ICMP (internet control message protocol)
|__> allows network-connected devices to send warning & error messages
|__> also connect request failures/success, or unavailable
|__> not used to get/send data to/from devices on the network
